## 报告总体结构

### 1.1 标准四段式

```
┌─────────────────────────────────────────────┐
│  一、产业特征    → 数据驱动的现状全景        │
│  二、核心趋势    → 时间维度的变化方向        │
│  三、存在问题    → 数据暴露的风险与短板      │
│  四、解决方案    → 可落地的政策建议          │
└─────────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **先数据后结论** | 每个结论必须有数据表格支撑，杜绝"拍脑袋" |
| **一表一洞察** | 每张表格配一段洞察文字，不超过 3 句话 |
| **可执行建议** | 解决方案必须有操作主体、时间节点、量化目标 |
| **数据透明** | 标明统计口径、数据来源、清洗方法、已知限制 |

## 四段式模板详解

### 第一段: 产业特征

**目的**：用数据刻画"这个区域的产业长什么样"

#### 必备模块

**1.1 总量概览**

| 指标 | 数值 | 同比 | 全市占比 |
|------|:----:|:----:|:--------:|
| 企业总数（四上口径） | `<N>` | `<±X%>` | `<Y%>` |
| 产值合计（亿元） | `<N>` | `<±X%>` | `<Y%>` |
| 制造业占比 | `<X%>` | — | — |
| 亿元以上企业数 | `<N>` | `<±X>` | — |

**1.2 梯队金字塔**

```
        🔴 巨头 (≥100亿): N家, 产值占比 X%
      🟠 龙头 (10-100亿): N家, 产值占比 X%
    🟡 骨干 (1-10亿): N家, 产值占比 X%
  🟢 中小 (<1亿): N家, 产值占比 X%
```

**1.3 产业集群排名**

| 集群 | 企业数 | 产值(亿) | 增速 | 小巨人 | 国高 | LQ |
|------|:------:|:--------:|:----:|:------:|:----:|:---:|
| `<集群1>` | N | N | ±X% | N | N | ★★ |
| `<集群2>` | N | N | ±X% | N | N | ★ |
| ... | | | | | | |

**SQL 查询模板**：

```sql
-- 产业集群 TOP10 排名
SELECT 
    cluster_name,
    COUNT(*) AS enterprise_count,
    ROUND(SUM(output_clean), 1) AS total_output_yi,
    ROUND((SUM(output_clean) - SUM(prev_output_clean)) / 
          NULLIF(SUM(prev_output_clean), 0) * 100, 1) AS growth_pct,
    COUNT(*) FILTER (WHERE xiaojuren IS NOT NULL) AS xiaojuren_count,
    COUNT(*) FILTER (WHERE gaoxin = '1') AS gaoxin_count
FROM cluster_enterprises
GROUP BY cluster_name
ORDER BY total_output_yi DESC;
```

**1.4 街道格局**

| 街道 | 企业数 | 产值(亿) | 主导产业TOP2 | 多样性指数 |
|------|:------:|:--------:|-------------|:----------:|
| `<街道A>` | N | N | 电子/装备 | 0.85 |
| ... | | | | |

**1.5 产业集中度**

- CR5（TOP5企业产值占比）
- HHI（赫芬达尔-赫希曼指数）→ 行业竞争格局
- 多样性指数（1-HHI 归一化）

### 第二段: 核心趋势

**目的**：从时间维度揭示"产业往哪走"

#### 必备模块

**2.1 增速分化分析**

| 区间 | 企业数 | 占比 | 代表企业（脱敏） |
|------|:------:|:----:|-----------------|
| >100% 爆发 | N | X% | `<企业类型>` |
| 30-100% 高增 | N | X% | `<企业类型>` |
| 10-30% 快速增长 | N | X% | `<企业类型>` |
| 0-10% 稳健 | N | X% | `<企业类型>` |
| -30%~0% 下滑 | N | X% | `<企业类型>` |
| <-30% 断崖 | N | X% | `<企业类型>` |

**SQL 查询**：

```sql
SELECT 
    CASE 
        WHEN growth_pct > 100 THEN '>100%爆发'
        WHEN growth_pct BETWEEN 30 AND 100 THEN '30-100%高增'
        WHEN growth_pct BETWEEN 10 AND 30 THEN '10-30%快速增长'
        WHEN growth_pct BETWEEN 0 AND 10 THEN '0-10%稳健'
        WHEN growth_pct BETWEEN -30 AND 0 THEN '-30%~0%下滑'
        ELSE '<-30%断崖'
    END AS growth_tier,
    COUNT(*)
FROM enterprises
WHERE is_yi_enterprise = true AND growth_pct IS NOT NULL
GROUP BY growth_tier
ORDER BY MIN(growth_pct);
```

**2.2 新亿元企业进出**

| 类型 | 企业数 | 平均产值(亿) | 主要集群 |
|------|:------:|:------------:|----------|
| 新晋亿元 | N | N | `<集群名>` |
| 退出亿元 | N | N | `<集群名>` |

**2.3 企业年龄-增速关系**

| 年龄段 | 企业数 | 平均增速 | 亿元占比 |
|--------|:------:|:--------:|:--------:|
| <5年 | N | ±X% | X% |
| 5-10年 | N | ±X% | X% |
| 10-20年 | N | ±X% | X% |
| >20年 | N | ±X% | X% |

> **关键判断**：如果老企业增速高于新企业，说明产业根基深厚；反之则需警惕"创新断层"。

**2.4 产值预测（趋势外推）**

```
基期产值 × (1 + CAGR)^5 = 预测值
CAGR = (末期/基期)^(1/年数) - 1
```

| 集群 | 基期(亿) | 末期(亿) | CAGR | 预测次年(亿) |
|------|:--------:|:--------:|:----:|:------------:|
| `<集群A>` | N | N | X% | N |

### 第三段: 存在问题

**目的**：从数据中挖掘风险信号，不做回避

#### 必备模块

**3.1 头部集中风险**

> CR5 > X% 的集群存在"一家独大"风险——如果链主企业外迁，集群可能崩塌。

**3.2 传统赛道天花板**

> 增速 < 0% 的集群面临需求萎缩/技术替代，需规划转型路线。

**3.3 新赛道缺位**

> 区域在新兴产业（如低空经济、AI 硬件、储能）中的企业数和产值占比偏低。

**3.4 断崖预警**

```
🔴 断崖预警名单（增速 < -30%）：
- <企业类型A>: N家, 产值合计 N亿, 主流集群 <集群名>
- <企业类型B>: ...
```

**3.5 空间失衡**

> 产出率方差过大：高产园区（>X亿/ha）与低效园区（<Y亿/ha）并存，土地利用效率悬殊。

**SQL 查询**：

```sql
-- 低效园区识别（TOD圈内·产出率低·面积大）
SELECT park_name, street, output_per_ha, park_area_ha,
       total_output_yi, enterprise_count, nearest_metro_m
FROM parks
WHERE output_per_ha < 0.5
  AND total_output_yi > 0
  AND park_area_ha > 1
  AND nearest_metro_m < 2000
ORDER BY output_per_ha ASC;
```

**3.6 创新-产业脱节**

> 创新载体500m内园区产出率（X亿/ha）vs 无创新覆盖园区（Y亿/ha），两者差异不显著→创新溢出效应不足。

## 测算依据方法论

> **核心原则**：无依据不拍脑袋。每个规划目标指标的数值必须有推演过程和公式支撑。

### 方法一: 份额法

**公式**：
```
目标值 = 上级目标 × 本区占全市份额 + 现状基数
```

**适用条件**：
- 上级规划有明确的量化目标
- 本区在全市的份额相对稳定（近3年波动 < 5pp）
- 产业结构与全市趋同

**示例**（脱敏）：
```
指标：<区域名>工业总产值2028年目标

已知：
- 全市2028年目标 = <T> 万亿
- 本区近3年占全市份额均值 = <S>%
- 现状基数 = <C> 亿

推演：
目标 = <T> × <S>% = <N> 亿
考虑增速惯性调整：<N> × (1 + 调整系数) = <N'> 亿
```

**标注格式**：
> 📐 方法：份额法 | 公式：`<公式>` | 基期：`<年份>年<数值>` | 可行性：🟢 | 风险：份额波动可能 ±X%

### 方法二: 趋势外推

**公式**：
```
预测值 = 基期值 × (1 + 年均增速)^预测年数
CAGR = (末期值 / 基期值)^(1/年数) - 1
```

**适用条件**：
- 有连续 3 年以上历史数据
- 增长趋势稳定（R² > 0.7）
- 无重大政策或技术颠覆预期

**示例**（脱敏）：
```
指标：<区域名>规上工业产值 2028 年预测

基期数据：
- 2021: <A> 亿
- 2022: <B> 亿
- 2023: <C> 亿
- CAGR(2021-2023) = <X>%

推演：
2028 = <C> × (1 + <X>%)^5 = <N> 亿
```

**保守/乐观情景**：
| 情景 | CAGR | 2028预测(亿) | 概率 |
|------|:----:|:------------:|:----:|
| 乐观 | +<X>% | <N1> | 20% |
| 基准 | +<Y>% | <N2> | 60% |
| 保守 | +<Z>% | <N3> | 20% |

**标注格式**：
> 📈 方法：趋势外推 | CAGR: `<X>%` | 基期: `<年份>` | R²: `<值>` | 可行性：🟡 | 风险：增速拐点不确定性

### 方法三: 梯队差

**公式**：
```
区级目标 = 上级目标 - 合理区级折扣
```

**适用条件**：
- 上级有明确的量化目标
- 区级应有合理梯度（通常是 3-8 个百分点）
- 本区与上级的经济发展水平存在显著差异

**折扣设定依据**：

| 本区相对水平 | 折扣(pp) | 说明 |
|:-----------:|:--------:|------|
| 高于全市均值 | 0-3 | 领跑区，折扣小 |
| 接近全市均值 | 3-5 | 持平区，标准折扣 |
| 低于全市均值 | 5-8 | 追赶区，折扣大 |
| 显著落后 | 8-12 | 后发区，实事求是 |

**标注格式**：
> 🪜 方法：梯队差 | 上级目标: `<值>` | 折扣: `<X>pp` | 依据: `<理由>` | 可行性：🟡 | 风险：需持续投入

### 方法四: 保守底线

**公式**：
```
保守目标 = 取显著低于线性推导的值（通常为线性推导的 60%-80%）
```

**适用条件**：
- 数据质量差，历史趋势不稳定
- 涉及高不确定性指标（如独角兽数、上市企业数）
- 新兴产业，缺乏历史参照

**示例**（脱敏）：
```
指标：<区域名>独角兽企业数 2028 年目标

现状：<N> 家
线性推导：每年新增 <M> 家 × 5年 = <N+5M> 家

保守底线（取60%）：<N+5M×0.6> 家
理由：独角兽产生具有高度不确定性，受资本市场周期影响大
```

**标注格式**：
> 🛡️ 方法：保守底线 | 线性推导: `<值>` | 折减系数: `0.6` | 理由: `<理由>` | 可行性：🔴 | 风险：如有系统性突破可超预期

### 四法选用决策树

```
上级有明确目标？
  ├── 是 → 本区份额稳定？
  │         ├── 是 → 【份额法】
  │         └── 否 → 本区与上级差距大？
  │                    ├── 是 → 【梯队差】
  │                    └── 否 → 看历史数据
  └── 否 → 有3年以上历史数据？
              ├── 是 → 趋势稳定？
              │         ├── 是 → 【趋势外推】
              │         └── 否 → 【保守底线】
              └── 否 → 【保守底线】
```

## 指标交叉验证模板

当单一数据源不可靠时，启动多路径交叉验证，取中值或合理区间。

### 场景1: 全员劳动生产率

| 路径 | 方法 | 数据来源 |
|------|------|----------|
| A·参保占比法 | 全市从业估数 × 本区参保占比 | `参保人` 字段 |
| B·X亿企业外推法 | X亿以上参保 ÷ 覆盖率 + X亿以下估数 | 企业底表 |
| C·产值占比法 | 全市从业估数 × 本区产值占比 | 规划原文 |

> **输出**：三路径结果 → 取中值作为推定从业人员 → 产值 ÷ 从业人员 = 劳动生产率

### 场景2: 研发投入强度

| 路径 | 方法 | 数据来源 |
|------|------|----------|
| A·全社会R&D反推 | 全社会R&D ÷ GDP | 统计年鉴 |
| B·全市反推 | 全市R&D强度 × 本区调整系数 | 规划原文 |
| C·高企反推 | 高企研发投入合计 ÷ 产值 | 企业底表 |
| D·目标梯度差 | 上级目标 - 合理折扣 | 规划原文 |
| E·专利产出反推 | 专利数 × 单位专利研发投入 | 知识产权数据 |

> **输出**：五路径取交集 → 合理区间 → 推荐值

### 场景3: 工业设计中心目标

分层推导法：
```
国家级：现有N家 + 候选M家 × 通过率X% = 目标N1
省级：  现有N家 + 候选M家 × 通过率X% = 目标N2
市级：  现有N家 + 候选M家 × 通过率X% = 目标N3
总计：  N1 + N2 + N3
```

## 数据质量分级体系

### 分级标准

| 等级 | 标记 | 含义 | 判定标准 |
|:----:|:----:|------|----------|
| 🟢 | 有实据 | 已经过交叉验证，可直接引用 | 多数据源一致 / 官方发布 |
| 🟡 | 推估值 | 基于合理方法推算，可引用但需标注 | 单一数据源 + 合理方法论 |
| 🔴 | 需核实 | 数据质量存疑，引用时必须注明风险 | 数据缺失 / 口径不明确 / 单源未验证 |

### 标注模板

```markdown
### 数据质量说明

| 指标 | 数值 | 分级 | 数据来源 | 计算方法 | 注意事项 |
|------|:----:|:----:|----------|----------|----------|
| 规上工业产值 | <N>亿 | 🟢 | `<数据源>` | 直接汇总 | 四上企业口径 |
| 研发投入强度 | <X>% | 🟡 | `<数据源>` | 五路径交叉取中值 | 研发投入列为区间格式 |
| 独角兽企业数 | <N>家 | 🔴 | `<数据源>` | 榜单匹配 | 口径可能不完整 |
```

### 常见数据质量问题与处理

| 问题 | 处理方式 | 标注 |
|------|----------|:----:|
| 产值字段含非数字字符 | 正则清洗后 CAST | 🟡 |
| 参保字段含 `-` 等脏数据 | 正则过滤 `^[0-9]+$` | 🟡 |
| 研发投入为区间格式（如"0.03-0.05"） | 取区间中值 | 🟡 |
| 分类标签为连续得分（0-1）而非布尔值 | 设阈值（≥0.5）后使用 | 🟡 |
| 集团用电数据为复制值 | 标注"数据待核" | 🔴 |
| 空置面积数据极小 | 标注"数据质量待核，可能不完整" | 🔴 |
| 企业底表非全量（四上口径） | 在报告首段标明口径 | 🟡 |

2. **一表一洞察**：每张表格配一段简短洞察
3. **标明口径**：首次提及数据时注明统计口径（四上/全量/抽样）
4. **数据可溯源**：标注数据来源表名、查询时间
5. **结论可执行**：方案段避免"要加强""要提升"等空泛表述，必须有主体、动作、目标

### 已知陷阱速查

| 陷阱 | 症状 | 修复 |
|------|------|------|
| 四上企业口径 | "零产值园区"占比高 | 改用"无四上企业入园" |
| PG ROUND 报错 | `function round(double, int) does not exist` | 加 `::numeric` 转换 |
| ST_DWithin 单位 | 缓冲区覆盖全球 | 加 `::geography` |
| 分类标签非布尔 | 筛选结果过少 | 先 `DISTINCT` 看分布再设阈值 |
| 研发投入区间格式 | CAST 报错 | 正则拆区间取中值 |
| 列名含中文 | shell 传参报 syntax error | 用 `psql -f script.sql` |
| 密码明文 | 终端历史泄露 | 只用 `.pgpass` 免密连接 |
